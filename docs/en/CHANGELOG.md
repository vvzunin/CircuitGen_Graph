# Changelog

All notable changes to **CircuitGen Graph** are documented in this file.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and releases use [Semantic Versioning](https://semver.org/) — see [Versioning.md](Versioning.md).

**Русский:** [CHANGELOG.md](../ru/CHANGELOG.md)

## [Unreleased]

### Added

- Verilog import into the graph (`GraphReader`, `OrientedGraph::readVerilog`, **lorina** submodule).
- External Verilog file binding for subgraph vertices; `parameter` / `localparam` parsing.
- Synology NAS documentation deploy from CI.
- Topic guides: GraphML, Verilog import, subgraphs, TestbenchGenerator (ru/en).
- Documentation index entries for `FUNCTIONALITY`, `Sequential`, `MEMORY_MANAGEMENT`, and new guides.
- Functionality overview in `docs/ru|en/FUNCTIONALITY.md`.
- Bus support and `oneBitVerilog` for gates; sequential refactoring.

### Changed

- Unified logging via `CG_LOG_*` macros and **easylogging++**.
- Full English `docs/en/Sequential.md`; corrected `affre` port order in examples.
- Legacy `docs/HACKING.md` and `docs/BUILDING.md` replaced with redirects to `docs/ru|en/`.
- Changelog moved to `docs/ru|en/CHANGELOG.md`.

### Fixed

- `parameter` / `localparam` emission in hierarchical Verilog generation.
- CI/docs: NAS deploy (Auth v3, credentials, DSM volume paths).
- Testbench tests: unique temp files; coverage and **fmt** fetch.
- GraphML/Subgraphs docs aligned with Classic GraphML behavior and `unrollGraph()`.

## How to update

When releasing `vX.Y.Z`:

1. Move **Unreleased** items into `## [X.Y.Z] - YYYY-MM-DD`.
2. Tag: `git tag vX.Y.Z`.
3. Version hint: `bash scripts/release/suggest-next-version.sh`.

Keep **both** `docs/ru/CHANGELOG.md` and `docs/en/CHANGELOG.md` in sync.
